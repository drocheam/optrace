5.3. Geometric Features#

5.3.1. TiltedSurface Equation#

\[\begin{split}\text{normal vector:}~~~~ \vec{n} &= (n_x, n_y, n_z)\\ \text{surface center vector:}~~~~ \vec{q} &= (x_0, y_0, z_0)\\ \text{point on surface:}~~~~ \vec{p} &= (x, y, z)\\\end{split}\]

point normal equation for a plane:

(5.30)#\[(\vec{p} - \vec{q})\cdot \vec{n} = 0\]

being equivalent to

(5.31)#\[(x - x_0) \cdot n_x + (y- y_0) \cdot n_y + (z-z_0)\cdot n_z = 0\]

can be rearranged to the surface function for \(n_z \neq 0\):

(5.32)#\[z(x, y) = z_0 - (x - x_0) \cdot \frac{n_x}{n_z} - (y- y_0) \cdot \frac{n_y}{n_z}\]

5.3.2. Analytical Hit Detection#

5.3.2.1. Intersection Ray with xy-Plane#

\[\begin{split}\text{surface normal vector:}~~~~ \vec{n} &= (n_x, n_y, n_z)\\ \text{surface center vector:}~~~~ \vec{q} &= (x_0, y_0, z_0)\\ \text{point on ray or surface:}~~~~ \vec{p} &= (x, y, z)\\ \text{ray support vector:}~~~~ \vec{p_0} &= (x_{0p}, y_{0p}, z_{0p})\\\end{split}\]

surface point normal equation:

(5.33)#\[(\vec{p} - \vec{q})\cdot \vec{n} = 0\]

ray equation in dependence of ray paramter \(t\):

(5.34)#\[\vec{p} = \vec{p_0} + \vec{s} \cdot t\]

inserting these equations into each other leads to

(5.35)#\[(\vec{p_0} + \vec{s}\cdot t_\text{h} - \vec{q}) \cdot \vec{n} = 0\]

rearranging gives us the ray parameter for the hit point \(t_\text{h}\):

(5.36)#\[t_\text{h} = \frac{(\vec{q} - \vec{p_0})\cdot \vec{n}}{\vec{s} \cdot \vec{n}}\]

which can be inserted into the ray equation to get the hit point

5.3.2.2. Intersection Ray with a Conic#

(5.37)#\[\begin{split}\text{Ray support vector:}~~~~ \vec{p} &= (p_x, p_y, p_z)\\ \text{Ray direction vector:}~~~~ \vec{s} &= (s_x, s_y, s_z)\\ \text{Center of surface:}~~~~ \vec{q} &= (x_0, y_0, z_0)\end{split}\]
(5.38)#\[p_z + s_z t = z_0 + \frac{\rho r^2}{1 + \sqrt{1-(k+1)\rho^2 r^2}}\]

with

(5.39)#\[r^2 = (p_x + s_x t - x_0)^2 + (p_y+s_y t - y_0)^2\]

Some work in rearanging leads to

(5.40)#\[A t^2 + 2 B t + C = 0\]

with

(5.41)#\[\begin{split}A &= 1 + k s_z^2\\ B &= o_x s_x + o_y s_y - \frac{s_z}{\rho} + (k+1) o_z s_z\\ C &= o_x^2 + o_y^2 - 2\frac{o_z}{\rho} + (k+1) o_z^2\\ \vec{o} &= \vec{p} - \vec{q} = (o_x, o_y, o_z)\end{split}\]

The solutions for \(t\) are

(5.42)#\[\begin{split}t = \begin{cases} \frac{-B \pm \sqrt{B^2 -CA}}{A} & \text{for}~~ A \neq 0, ~~ B^2 - CA \geq 0 \\ -\frac{C}{2B} & \text{for}~~ A = 0, ~~B \neq 0\\ \{\mathbb{R}\} & \text{for}~~ A = 0, ~~B = 0, ~~C = 0\\ \emptyset & \text{else} \end{cases}\end{split}\]

5.3.2.3. Surface Extension#

5.3.3. Numerical Hit Detection#

5.3.3.1. Bound Calculation#

5.3.3.2. Intersections#

\[\begin{split}\text{Ray support vector:}~~~~ \vec{p_0} &= (p_x, p_y, p_z)\\ \text{Ray direction vector:}~~~~ \vec{s} &= (s_x, s_y, s_z)\\ \text{Point on Ray:}~~~~ \vec{p_t} &= (x_t, y_t, z_t)\\\end{split}\]

Ray line equation depending on ray parameter \(t\):

(5.43)#\[\vec{p_t}(t)=\vec{p}_{0}+t \cdot \vec{s}\]

Cost function \(G\) with surface function \(f\):

(5.44)#\[G(t)=z_{t}-f\left(x_{t}, y_{t}\right)\]

The parameters \(x_t,y_t,z_t\) can be determined from equation (5.43). For the position determination of the hit, the root of this scalar function \(G\) must now be found. Typical optimization algorithms are suitable for this purpose.

However, these have the disadvantage that they do not have a guaranteed convergence. Therefore, the ray tracer uses the Regula-Falsi method. This is a simple iterative method, which is guaranteed to converge superlinearly with a slight modification. The prerequisite for the procedure is, however, that an interval with a root is known. Since the minimum and maximum extent of the surface in the z-direction in the raytracer is known, this criterion is given, because a hit can only occur within this range. The method basically works by trying to shrink the interval including the function root in every iteration. A well-written explanation can be found in [1].

Now, in some cases the interval may hardly decrease in size from one iteration to the next. To prevent slow convergence the procedure is therefore extended to the so called Illinois algorithm, which is explained in [2].

The implementation in optrace differs only in parallelizing the optimization and only iterating the next step with not already converged rays.

5.3.4. Normal Calculation#

5.3.4.1. General#

Equation for a normal vector: [3]

(5.45)#\[\begin{split}\vec{n_0} = \begin{pmatrix} -\frac{\partial z}{\partial x}\\ -\frac{\partial z}{\partial y}\\ 1\\ \end{pmatrix}\end{split}\]

Needs to be normalized using

(5.46)#\[\vec{n} = \frac{\vec{n_0}}{|| \vec{n_0} ||}\]

5.3.4.2. Numerical#

(5.47)#\[\begin{split}\vec{n_0} = \begin{pmatrix} z(x - \varepsilon, ~y) - z(x + \varepsilon, ~y)\\ z(x, ~y - \varepsilon) - z(x, ~y + \varepsilon)\\ \varepsilon\\ \end{pmatrix}\end{split}\]

Needs to be normalized using

(5.48)#\[\vec{n} = \frac{\vec{n_0}}{|| \vec{n_0} ||}\]

See {} for notes on choosing \(\varepsilon\).

5.3.4.3. Plane#

(5.49)#\[\begin{split}\vec{n} = \begin{pmatrix} 0\\ 0\\ 1\\ \end{pmatrix}\end{split}\]

5.3.4.4. Conic#

The derivative of the conic function is

(5.50)#\[m = \tan{\alpha} = \frac{\text{d}z(r)}{\text{d}r} = \frac{\rho r}{\sqrt{1 - (k+1)\rho^2 r^2}}\]
(5.51)#\[n_r = -\sin{\alpha} = -\frac{m}{\sqrt{m^2+1}} = -\frac{\rho r}{\sqrt{1- k\rho^2 r^2}}\]
(5.52)#\[\begin{split}n_x &= n_r \cos \phi\\ n_y &= n_r \sin \phi\\ n_z &= \sqrt{1- n_r^2}\end{split}\]
(5.53)#\[\begin{split}\vec{n} = \begin{pmatrix} n_x\\ n_y\\ n_z\\ \end{pmatrix}\end{split}\]

5.3.4.5. Sphere#

With \(k=0\) and \(\rho := \frac{1}{R}\) the conic normal simplifies to

(5.54)#\[\begin{split}\vec{n} = \begin{pmatrix} -\rho r \cos \phi \\ -\rho {}r \sin \phi\\ \sqrt{ 1 - \rho^2 r^2}\\ \end{pmatrix}\end{split}\]

5.3.4.6. Asphere#

The radial derivative of an asphere is

(5.55)#\[n_r = \frac{\rho r}{\sqrt{1 - (k+1)\rho^2 r^2}} + \sum_{i=1}^{m} 2i \cdot a_i \cdot r^{2i - 1}\]

This radial component needs to be rotated around the center by angle the positional angle \(\phi\). According to the general normal calculation the normal is then

(5.56)#\[\begin{split}\vec{n_0} = \begin{pmatrix} - n_r \cos \phi\\ - n_r \sin \phi\\ 1\\ \end{pmatrix}\end{split}\]

This needs to be normalized using

(5.57)#\[\vec{n} = \frac{\vec{n_0}}{|| \vec{n_0} ||}\]

5.3.5. Numerical Differentiation#

5.3.5.1. Central first derivative#

For a first derivative of the form

(5.58)#\[f'(x) = \frac{f(x+\varepsilon) - f(x-\varepsilon)}{2 \varepsilon}\]

the optimal step width is [4]:

(5.59)#\[\varepsilon_\text{o} = \sqrt[3]{3 \varepsilon_\text{f} \left| \frac{f(x)}{f^{(3)}(x)} \right|}\]

with \(\varepsilon_\text{f}\) being the machine precision for the used floating type. Expecting mostly spherical surfaces, the main function component is \(x^2\). Higher polynomial orders are less prominent, so one valid assumption can be \(\left| \frac{f(x)}{f^{(3)}(x)} \right| = 50\). While this might be different for every function, due to the forth root a quotient being 1000 times larger only leads to a change of around factor \(10\) in \(\varepsilon_\text{0}\).

With \(\varepsilon_\text{f} \approx 2.22\cdot 10^{-16}\) for a 64bit floating point number, we get \(\varepsilon_\text{o} \approx 3.22 \cdot 10^{-5}\). Optrace units are given in millimeters, so this is equivalent to a value of \(32.2\,\) nm.

5.3.5.2. Step Width Selection#

Not only differences in \(f\) need to be representable, but \(x+\varepsilon\) needs to be different from \(x\). For this it must be ensured, that \(x+ \varepsilon > x (1 + \varepsilon_\text{f})\) for every coordinate \(x\) on the surface. With \(R\) being the largest absolute distance on the surface the minimal bound is

(5.60)#\[\varepsilon_\text{n} = R ~\varepsilon_\text{f}\]

It is recommended to center the surface at \(x=0\) so \(R\) is minimal. This only works if the surface is centered beforehand, shifting afterwards also ruins numerical precision.

The finally chosen step width is the higher one:

(5.61)#\[\varepsilon = \max (\varepsilon_\text{o}, ~\varepsilon_\text{n})\]

5.3.6. Sphere Projections#

The relative distance to center and the z-position of the other sphere end are

(5.62)#\[\begin{split}r &= \sqrt{(x-x_0)^2 + (y - y_0)^2}\\ z_m &= z_0 + R\end{split}\]

Equidistant

Adapted version of [5].

(5.63)#\[\begin{split}\theta &= \arctan\left(\frac{r}{z-z_m}\right)\\ \phi &= \text{arctan2}(y-y_0, ~x-x_0)\\\end{split}\]

The projected coordinates are then

(5.64)#\[\begin{split}x_p &= -\theta \cdot \text{sgn}(R) \cos(\phi)\\ y_p &= -\theta \cdot \text{sgn}(R) \sin(\phi)\\\end{split}\]

Orthographic

The hit coordinates \(x\) and \(y\) are kept as is. Related: [6].

Stereographic

Adapted version of [7].

(5.65)#\[\begin{split}\theta &= \frac{\pi}{2} - \arctan\left(\frac{r}{z-z_m}\right)\\ \phi &= \text{arctan2}(y-y_0, ~x-x_0)\\ r &= 2 \tan\left(\frac{\pi}{4} - \frac{\theta}{2}\right)\\\end{split}\]

The projected coordinates are then

(5.66)#\[\begin{split}x_p &= -r \cdot \text{sgn}(R) \cos(\phi)\\ y_p &= -r \cdot \text{sgn}(R) \sin(\phi)\\\end{split}\]

Equal-Area

Adapted version of [8].

(5.67)#\[\begin{split}x_r = \frac{x - x_0} {\lvert R \rvert}\\ y_r = \frac{y - y_0} {\lvert R \rvert}\\ z_r = \frac{z - z_m} {R}\\\end{split}\]

The projected coordinates are then

(5.68)#\[\begin{split}x_p = \sqrt{\frac{2}{1-z_r} x_r}\\ y_p = \sqrt{\frac{2}{1-z_r} y_r}\\\end{split}\]

5.3.7. Random Sampling#

5.3.7.1. Point#

Since a point only has one position \(x_0,~y_0,~z_0\), all random values have these coordinates.

5.3.7.2. Line#

Generate a uniform variable \(\mathcal{U}_\text{[-R,R]}\) with \(R\) being half the length of the line.

The random cartesian coordinates are then

(5.69)#\[\begin{split}x =&~ x_0 + \mathcal{U}_{[-R,R]} \cos \alpha\\ y =&~ y_0 + \mathcal{U}_{[-R,R]} \sin \alpha\\ z =&~ z_0\end{split}\]

\(x_0,~y_0,~z_0\) are the central coordinates of the line and \(\alpha\) is its angle relative to the x-axis.

5.3.7.3. Rectangle#

For uniform random positions on a rectangular surface we need to generate two independent random uniform variables, where each range is the extent \([x_0,~x_1,~y_0,~y_1]\) of the rectangle.

The random cartesian coordinates are then

(5.70)#\[\begin{split}x =&~ \mathcal{U}_{[x_0,x_1]}\\ y =&~ \mathcal{U}_{[y_0,y_1]}\\ z =&~ z_0\end{split}\]

5.3.7.4. Ring#

An area element of a circle in polar coordinates can be represented as:

(5.71)#\[\text{d}A = \text{d}r ~\text{d}\phi\]

\(\text{d}\phi\) can be rewritten as circle segment

(5.72)#\[\text{d}A = \text{d}r ~\frac{2 \pi}{N} r\]

with \(N\) being the number of segments. Let us define a function \(r(u)\) which gives us radial values and its derivative outputs radial spacing values.

(5.73)#\[\text{d}A = r'(u) ~\frac{2 \pi}{N} r(u)\]

For uniformly sampled data, \(\text{d}A\) needs to be kept constant in regards to a uniform variable \(u\). This is equivalent to the condition \(\frac{\text{d}A}{\text{d}u} = 0\).

(5.74)#\[\frac{\text{d}A}{\text{d}u} = \frac{2\pi}{N} \frac{\text{d}}{\text{d}u} r'(u) r(u) = r''(u) r(u) + (r'(u))^2 = 0\]

Solutions of this non linear differential equation of second order are in the form of

(5.75)#\[r(u) = \sqrt{c_1 + c_2 u}\]

For convenience we set the constants to \(c_1 = 0, ~c_2=1\). For output values in \([r_i, ~R]\) the corresponding input values are then \([r^2_i, ~R^2]\). Rewriting \(r\) and \(u\) as random variables gives us:

(5.76)#\[\mathcal{R} = \sqrt{\mathcal{U}_{[r^2_\text{i}, R^2]}}\]

The polar angle is uniformly spaced

(5.77)#\[\Phi = \mathcal{U}_{[0, 2\pi]}\]

Resulting 3D positions are then

(5.78)#\[\begin{split}x =&~ x_0 + \mathcal{R} \cos \Phi\\ y =&~ y_0 + \mathcal{R} \sin \Phi\\ z =&~ z_0\end{split}\]

5.3.7.5. Circle#

Implemented as ring with \(r_\text{i} = 0\).

Related: [9].

5.3.8. Flipping and Rotation#

Flipping a surface is implemented as a 180 degree rotation around the x-axis.

This is equivalent to negating its relative shape \(z_r\) to an offset \(z_0\) and mirroring the y-component: \(z_0 + z_r(x, y) \Rightarrow z_0 - z(x, -y)\). For a surface with rotational symmetry this simplifies to \(z_0 + z_r(r) \Rightarrow z_0 - z(r)\)

Rotation is done by changing the accessing coordinates through a rotation of the coordinate system:

\(z(x, y) \Rightarrow z(x_0 + r \cos \alpha, y_0 + r \sin \alpha)\) with \(r = \sqrt{(x-x_0)^2 + (y - y_0)^2}\). Here \(x_0,\; y_0\) are the rotation center coordinates and \(\alpha\) is the rotation angle.

By simply changing the value of the rotation angle the surface values can be rotated without actually rotating the surface.
